# 절세추천 페이지 구현 계획

## 0. UI 기준

- 절세추천 페이지는 **메인화면을 레퍼런스**로 삼아 구현한다.
- 메인화면에서 이미 정리한 AppShell 비율, 카드 radius, 카드 간격, 폰트 토큰, 색상 토큰, 버튼 비율, 하단 내비게이션 감각을 유지한다.
- 새 UI를 독립적으로 예쁘게 만드는 것이 아니라, 사용자가 같은 앱 안의 다음 화면으로 인식할 수 있도록 통일감을 우선한다.
- 큰 카드/배너는 가능하면 기존 `AppCard`, `ui-*` typography class, `src/components/` 컴포넌트 구조를 재사용한다.
- 개별 요소마다 임의 `font-size`, `font-weight`, `color`, margin을 새로 찍지 않는다. 필요하면 기존 토큰/컴포넌트 규칙을 먼저 따른다.
- 그라데이션은 금지한다. 배경은 단색 또는 기존 토큰 기반 색만 사용한다.

## 1. 구현 대상

- 파일: `src/features/taxSaving/TaxSavingPage.jsx`
- 현재 placeholder를 실제 절세추천 화면으로 교체한다.
- AppShell은 `AppRoutes.jsx`에서 이미 감싸고 있으므로 페이지 파일은 본문 UI만 담당한다.

## 3. 데이터 원칙

- 화면 텍스트는 사용자가 제공한 레퍼런스 이미지의 문구를 하나하나 준수한다.
- 단, 기존 프로젝트 지침상 금액/혜택/정책 문구는 `mockdatas/`와 충돌하지 않아야 한다.
- 구현 전 `mockdatas/index.md`와 대조해 근거가 있는 항목은 source 주석 또는 데이터 필드로 연결한다.
- JSX 내부에 임의로 텍스트를 흩뿌리지 말고, 페이지 상단의 데이터 배열 또는 별도 구조로 모은다.
- 임의 금액, 임의 정책/공제명, 임의 설명은 추가하지 않는다.

## 4. 이미지 에셋 계획

필요한 투명 배경 PNG:

- 안경/렌즈
- 노트/교육비
- 병원
- 집
- 카드
- 젖병
- 하단 배너 문서+돋보기

저장 위치:

- `src/assets/tax-saving/`

규칙:

- 이미지 자체에는 테두리/그림자/프레임을 넣지 않는다.
- 카드 안의 연한 단색 이미지 배경 박스는 CSS로 처리한다.
- 이미지가 필요한데 없으면 임시 아이콘으로 대체하지 말고 투명 배경 이미지로 생성한다.

## 5. 컴포넌트 구조

새 재사용 컴포넌트는 `src/components/`에 만든다.

예상 컴포넌트:

- `src/components/TaxSavingCards.jsx`
  - `TaxSavingBenefitCard`
  - `TaxSavingSupportBanner`

페이지 파일:

- `src/features/taxSaving/TaxSavingPage.jsx`
  - 데이터 배열 정의
  - 페이지 레이아웃
  - 컴포넌트 호출

## 6. 레이아웃 계획

- AppShell 비율은 기존 `app-shell--standard`를 그대로 사용한다.
- 본문은 아래 순서로 구성한다.
  1. 상단 제목 영역
  2. 2열 추천 카드 그리드
  3. 하단 지원금 배너
- 카드 그리드는 메인화면 카드 간격과 유사한 rhythm을 유지한다.
- 카드 내부 구성:
  - 이미지
  - 제목
  - 금액
  - 태그
  - 우측 화살표
- 하단 배너 구성:
  - 좌측 텍스트/버튼
  - 우측 투명 배경 이미지
  - 단색 연파랑 배경

## 7. 스타일 규칙

- 메인화면에서 쓰는 카드/텍스트/버튼 규칙과 통일한다.
- 텍스트는 기존 `ui-*` typography class 우선 사용.
- 새 텍스트 계층이 정말 필요하면 global token/class를 먼저 추가하고, 요소 selector에 직접 폰트 값을 쓰지 않는다.
- 색상은 기존 `--color-text-*`, `--accent`, `--surface`, `--line` 계열 토큰을 우선 사용한다.
- 카드 간 외부 여백은 부모 grid/flex gap에서만 관리한다.
- 카드 root에 임의 margin을 넣지 않는다.
- 그라데이션 금지.

## 8. 링크 동작

- 추천 카드 클릭: `/${personaId}/deduction-detail`
- 하단 `내 맞춤 혜택 찾기`: 우선 `/${personaId}/policies`로 연결하는 것이 문맥상 적절하다.

## 9. 검증 계획

구현 후 아래를 실행한다.

- `npm run validate`
- 그라데이션 금지 검색:
  - `grep -R "linear-gradient\|radial-gradient\|conic-gradient" -n src dist/index.html dist/assets/*.css dist/assets/*.js || true`
- 레퍼런스 이미지 텍스트 누락 여부 수동 대조
- 메인/회원가입/AppShell 비율 회귀 확인
