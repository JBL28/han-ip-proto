# AI 페르소나 변경 페이지 구현 계획

## 0. UI 기준

- AI 페르소나 변경 페이지는 **메인화면의 AppShell 표준 비율과 카드/버튼 토큰**을 유지한다.
- 이 화면의 “페르소나”는 앱 라우팅 페르소나(`한버팀`, `김갓생`)가 아니라 챗봇 응답 스타일(`초보형`, `관심형`, `공부형`)이다.
- 라우팅 페르소나 선택 화면과 혼동되지 않도록 제목/설명/데이터 key를 분리한다.
- 선택 카드는 full-width 카드 또는 1열 리스트로 구성하고, 터치 영역은 48px 이상으로 둔다.
- 그라데이션은 금지한다.

## 1. 구현 대상

- 파일: `src/features/persona/PersonaSelectPage.jsx`
- 현재 landing/persona selection 용도와 충돌하지 않게, AppShell 내부 `/:personaId/persona`에서는 챗봇 페르소나 변경 화면으로 동작하게 한다.
- 데이터:
  - 챗봇 페르소나 정의는 `mockdatas/index.md > 챗봇 페르소나`
  - 앱 라우팅 페르소나는 `src/routes/routeConfig.js`의 `personas`
- 예상 컴포넌트:
  - `src/components/AIPersonaCards.jsx`
    - `AIPersonaOptionCard`
    - `AIPersonaConfirmBar`
- 와이어프레임: `wireframes/AI페르소나변경.png`

## 2. 텍스트 준수 목록

와이어프레임의 제목, 설명, 페르소나명, 수준 설명, CTA 문구는 구현 직전 다시 수동 대조한다. 챗봇 페르소나 문구는 아래 정의를 그대로 사용한다.

### 상단 후보

- `AI 페르소나 변경`
- `어떤 방식으로 설명해드릴까요?`
- `내게 맞는 설명 스타일을 선택해요`

### 챗봇 페르소나 3개

1. `초보형`
   - 정의: `아예 모르는사람(유치원 수준)`
   - 설명: `쉬운 설명`
2. `관심형`
   - 정의: `들어는 봤지만 (관심있지만) 자세히 모름(학생 수준)`
   - 설명: `요약 + 추천`
3. `공부형`
   - 정의: `약간의 정보를 알고 있고, 배울 의지가 있는 사람(성인 수준)`
   - 설명: `분석 + 전략`

### 페르소나별 기본값

- 한버팀: `관심형`
  - source: `mockdatas/index.md > 한버팀 > 챗봇`
  - 이유 문구: `어려운 행정 용어를 빼고 3줄 요약 + 맞춤형 추천 제공`
- 김갓생: `초보형`
  - source: `mockdatas/index.md > 김갓생 > 챗봇`
  - 이유 문구: `세금, 연말정산 1도 모르는 쌩초보를 위한 초간단 비유와 설명`

### CTA 후보

- `선택 완료`
- `적용하기`
- `챗봇으로 돌아가기`

최종 CTA 문구는 와이어프레임 문구를 우선한다.

## 3. 데이터 원칙

- 초보형/관심형/공부형 설명은 `mockdatas/index.md`의 정의를 그대로 사용한다.
- 앱 라우팅 페르소나와 챗봇 페르소나를 혼동하지 않도록 key를 분리한다.
  - 앱 페르소나 storage: `hanip.selectedPersona`
  - 챗봇 페르소나 storage 제안: `hanip.chatPersonaByPersona`
- 선택 상태는 localStorage 또는 React state로 저장하되 실제 서버 저장은 없다.
- 저장 실패 가능성을 고려해 localStorage 접근은 try/catch로 감싼다.
- 와이어프레임에 없는 추가 레벨/성격/설명은 만들지 않는다.

### 데이터 구조 제안

```js
const aiPersonas = [
  {
    id: 'beginner',
    name: '초보형',
    definition: '아예 모르는사람(유치원 수준)',
    description: '쉬운 설명',
    source: 'mockdatas/index.md > 챗봇 페르소나'
  },
  ...
]
```

## 4. 이미지 에셋 생성 계획

필요한 투명 배경 PNG:

- 초보형 아바타/아이콘
- 관심형 아바타/아이콘
- 공부형 아바타/아이콘
- 상단 챗봇/설정 오브젝트가 와이어프레임에 있으면 별도 생성

저장 위치:

- `src/assets/ai-persona/`

파일명:

- `persona-beginner.png`
- `persona-interested.png`
- `persona-study.png`
- `persona-settings.png`

생성 프롬프트 공통 규칙:

- 투명 배경 PNG, 텍스트 없는 캐릭터/오브젝트.
- 초보형: 쉬운 설명을 상징하는 작은 말풍선/기초 학습 느낌.
- 관심형: 요약과 추천을 상징하는 노트/체크 느낌.
- 공부형: 분석과 전략을 상징하는 돋보기/차트 느낌.
- 실제 글자, 숫자, 로고는 넣지 않는다.
- 이미지 배경/카드/선택 테두리는 CSS에서 처리한다.

## 5. 컴포넌트 구조

`PersonaSelectPage.jsx`

- `useParams()`로 현재 앱 `personaId` 확인.
- 기존 landing의 앱 페르소나 선택 UI와 분리한다.
- 본문 순서:
  1. 상단 제목/설명
  2. 현재 선택 안내 badge
  3. AI 페르소나 카드 리스트
  4. 선택 완료 CTA

`AIPersonaOptionCard`

- props: `{ option, selected, onSelect, imageSrc }`
- 전체 카드는 button으로 구현한다.
- 선택 상태는 `aria-pressed`, 시각적 border/background 단색으로 표시한다.

`AIPersonaConfirmBar`

- `선택 완료` 클릭 시 localStorage 저장 후 `/${personaId}/tax-chat`로 이동한다.
- 와이어프레임이 별도 완료 화면을 요구하지 않으면 즉시 이동한다.

## 6. 레이아웃 계획

- 1열 카드 리스트.
- 각 카드 내부: 이미지/페르소나명/정의/설명/선택 표시.
- 긴 정의 문구는 2줄 이상 줄바꿈 허용.
- CTA는 하단 고정 또는 본문 하단 배치. 고정 시 bottom nav와 겹치지 않게 한다.
- 카드 root margin 금지, 부모 gap 사용.

## 7. 링크 동작

- 선택 완료: `/${personaId}/tax-chat`
- 뒤로가기: `/${personaId}/my` 또는 browser history
- 앱 페르소나 변경이 필요한 경우에는 별도 landing 흐름으로 보내야 하며, 이 화면에서 한버팀/김갓생을 바꾸지 않는다.

## 8. 검증 계획

- `npm run validate`
- 그라데이션 금지 검색:
  - `grep -R "linear-gradient\|radial-gradient\|conic-gradient" -n src dist/index.html dist/assets/*.css dist/assets/*.js || true`
- 초보형/관심형/공부형 3개 문구가 source와 일치하는지 확인
- 선택 상태가 즉시 시각적으로 바뀌는지 확인
- 저장 후 `tax-chat`에서 선택값을 읽을 수 있는지 확인
- 앱 페르소나 localStorage와 챗봇 페르소나 localStorage가 충돌하지 않는지 확인
- 와이어프레임 텍스트 누락 수동 대조
