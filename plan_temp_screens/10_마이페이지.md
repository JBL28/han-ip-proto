# 마이페이지 구현 계획

## 0. UI 기준

- 마이페이지는 **메인화면의 AppShell 표준 비율**을 유지한다.
- 사용자 요약/혜택 카드/메뉴 리스트도 `AppCard` 기반으로 통일한다.
- 계정/설정 화면처럼 보이되, 이 프로토타입의 핵심은 누적 혜택과 페르소나/챗봇 설정 진입이다.
- 실제 계정, 로그아웃, 서버 저장은 구현하지 않는다.
- 그라데이션은 금지한다.

## 1. 구현 대상

- 파일: `src/features/my/MyPage.jsx`
- 현재 placeholder를 실제 마이페이지 화면으로 교체한다.
- 예상 컴포넌트:
  - `src/components/MyPageCards.jsx`
    - `MyProfileCard`
    - `BenefitTotalCard`
    - `MyMenuList`
    - `MyMenuItem`
- 와이어프레임: `wireframes/마이페이지.png`
- 관련 라우트:
  - `/${personaId}/my`
  - `/${personaId}/persona`
  - `/${personaId}/notifications`
  - `/${personaId}/policies`

## 2. 텍스트 준수 목록

와이어프레임의 프로필명, 메뉴명, 금액, 설명, 버튼 문구는 구현 직전 다시 수동 대조한다. 누적 혜택 금액은 `mockdatas/index.md`의 마이페이지 섹션 근거를 사용한다.

### 상단/프로필 후보

- `마이페이지`
- `한버팀`
- `김갓생`
- `내 정보`

### 누적 혜택 카드

한버팀:

- `지금까지 받은 혜택 금액`
- `올해 '한입'을 통해 확실하게 챙긴 누적 혜택 240만 원`
- 설명 근거: `작년 근로장려금 환급액 + 매월 아동수당 합산액`

김갓생:

- `지금까지 받은 혜택 금액`
- `취업 후 '한입'과 함께 차곡차곡 챙긴 누적 혜택 95만 원`
- 설명 근거: `작년 연말정산 월세 환급금 + K-패스 교통비 환급 등`

### 메뉴 후보

와이어프레임에 있는 항목만 최종 구현한다. 현재 설계상 필요한 후보:

- `스크랩한 정책`
- `AI 페르소나 변경`
- `알림`
- `개인정보 관리`
- `고객센터`
- `로그아웃`

단, `개인정보 관리`, `고객센터`, `로그아웃`은 mock UI이며 실제 기능을 만들지 않는다.

## 3. 데이터 원칙

- 페르소나 기본 정보는 `src/routes/routeConfig.js`의 `personas`를 사용한다.
- 누적 혜택 금액은 `mockdatas/index.md > 마이페이지`에서만 가져온다.
- 임의 금액, 임의 프로필 정보, 임의 메뉴 설명을 추가하지 않는다.
- 스크랩 정책 수, 신청 완료 수 같은 수치가 와이어프레임/source에 없으면 만들지 않는다.
- localStorage에 저장된 앱 페르소나(`hanip.selectedPersona`)와 챗봇 페르소나 설정은 구분한다.

### 데이터 구조 제안

```js
{
  personaId: 'hanbeoteam',
  profileName: '한버팀',
  profileDescription: '34세, 워킹맘 / 연소득 약 2,600만 원, 중소기업 사무직',
  benefitTitle: '지금까지 받은 혜택 금액',
  benefitAmount: '240만 원',
  benefitDescription: "올해 '한입'을 통해 확실하게 챙긴 누적 혜택",
  source: 'mockdatas/index.md > 한버팀 > 마이페이지'
}
```

## 4. 이미지 에셋 생성 계획

필요한 투명 배경 PNG:

- 한버팀 프로필 아바타
- 김갓생 프로필 아바타
- 누적 혜택 카드용 지갑/동전 오브젝트
- 스크랩 정책용 북마크/문서 오브젝트가 와이어프레임에 큰 이미지로 있으면 생성

저장 위치:

- `src/assets/my/`

파일명:

- `profile-hanbeoteam.png`
- `profile-kimgatsaeng.png`
- `benefit-wallet.png`
- `scrap-document.png`

생성 프롬프트 공통 규칙:

- 투명 배경 PNG, 텍스트 없는 캐릭터/오브젝트.
- 한버팀 아바타는 34세 워킹맘 맥락을 과장 없이 단정한 금융 앱 일러스트로 표현한다.
- 김갓생 아바타는 26세 사회초년생 맥락을 단정한 금융 앱 일러스트로 표현한다.
- 실제 인물 사진 느낌, 과도한 표정, 기관 로고, 텍스트 금지.
- 이미지 프레임과 배경은 CSS에서 단색으로 처리한다.

## 5. 컴포넌트 구조

`MyPage.jsx`

- `useParams()`로 `personaId` 확인.
- `getPersona(personaId)`와 마이페이지 mock data를 결합한다.
- 본문 순서:
  1. 상단 제목
  2. 프로필 카드
  3. 누적 혜택 카드
  4. 메뉴 리스트

`MyProfileCard`

- 이름, 설명, summary, 프로필 이미지.
- 앱 페르소나 변경이 필요하면 landing이 아니라 `/${personaId}/persona`는 챗봇 페르소나 변경이므로 혼동하지 않는다.

`BenefitTotalCard`

- 핵심 금액을 크게 표시하되 `ui-amount` 계열을 재사용한다.
- 설명은 `ui-body-muted` 계열.

`MyMenuList` / `MyMenuItem`

- 메뉴는 button 또는 Link.
- `AI 페르소나 변경`: `/${personaId}/persona`
- `알림`: `/${personaId}/notifications`
- `스크랩한 정책`: 1차 구현은 `/${personaId}/policies`로 연결하거나 mock disabled 상태.

## 6. 레이아웃 계획

- 1열 세로 stack.
- 프로필 카드와 혜택 카드는 메인 카드 폭과 동일하게 맞춘다.
- 메뉴 리스트는 full-width touch row, 각 row 높이 48px 이상.
- 마지막 메뉴가 bottom nav에 가려지지 않도록 bottom padding 확보.
- 카드 root에는 margin을 넣지 않고 부모 gap으로 간격을 관리한다.

## 7. 링크 동작

- `AI 페르소나 변경`: `/${personaId}/persona`
- `알림`: `/${personaId}/notifications`
- `스크랩한 정책`: `/${personaId}/policies` 또는 mock 상태
- `로그아웃`: 실제 로그아웃 없음. 와이어프레임에 반드시 있으면 alert/dialog 없이 mock button만 표시하거나 `/` 이동으로 제한한다.
