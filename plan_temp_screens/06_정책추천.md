# 정책추천 페이지 구현 계획

## 0. UI 기준

- 정책추천 페이지는 **메인화면을 레퍼런스**로 삼아 카드형 리스트 UI를 작성한다.
- AppShell 비율, typography token, color token, 카드 외곽 gap 규칙을 유지한다.
- 정책 카드는 `AppCard` 기반으로 만들고 카드끼리의 바깥 여백은 부모 gap에서만 관리한다.
- 정책추천은 “더 많은 지원금 찾기” 맥락의 화면이므로 절세추천 페이지 하단 배너와 자연스럽게 연결되어야 한다.
- 그라데이션은 금지한다. 필터 chip, 카드 배경, CTA는 단색만 사용한다.

## 3. 데이터 원칙

- 정책명/설명은 `mockdatas/*.xml` 및 `mockdatas/text/*`에 있는 복지로 데이터만 사용한다.
- `mockdatas/index.md`와 XML 표시명이 충돌하면 둘 다 데이터에 보존한다.
  - 예: `{ displayNameFromIndex, xmlServNm, sourceConflict: true }`
- UI에 노출할 필드:
  - `servNm`, `servDgst`, `jurMnofNm`, `jurOrgNm`, `sprtCycNm`, `srvPvsnNm`, `onapPsbltYn`, `rprsCtadr`, `lifeArray`, `intrsThemaArray`, `trgterIndvdlArray`
- 카드에 모든 필드를 다 보여주지 말고, 목록에서는 정책명/요약/태그/지원 형태만 보여준다.
- 상세 필드는 `policy-detail`에서 보여준다.

## 4. 이미지 에셋 생성 계획

필요한 투명 배경 PNG:

- 정책 추천 상단 일러스트: 문서+체크+돋보기
- 아이돌봄/보육 정책 오브젝트: 아이 돌봄 손/하트
- 문화이용권 오브젝트: 티켓/카드
- 직업훈련 오브젝트: 노트북+학습 카드
- 청년월세 오브젝트: 작은 집+동전
- 플로팅 챗봇 오브젝트가 와이어프레임에 있으면 챗봇 아이콘

저장 위치:

- `src/assets/policies/`

파일명:

- `policy-search.png`
- `child-care-policy.png`
- `culture-card.png`
- `job-training.png`
- `youth-rent.png`
- `policy-chatbot.png`

생성 프롬프트 공통 규칙:

- 투명 배경 PNG, 텍스트 없는 단일 오브젝트, 모바일 금융/공공서비스 앱에 맞는 부드러운 일러스트.
- 실제 정부 부처 로고, 복지로 로고, 카드번호, 한글 텍스트는 넣지 않는다.
- 이미지 자체에 배경 카드/테두리/그림자를 넣지 않는다.
- CSS에서 단색 원형/라운드 배경을 만든 뒤 이미지를 얹는다.

## 5. 컴포넌트 구조

`PoliciesPage.jsx`

- `useParams()`로 `personaId` 확인.
- `getPersonaPolicyData(personaId)` 형태로 페르소나별 목록 로딩.
- 본문 순서:
  1. 상단 제목/설명
  2. 카테고리 chip row(와이어프레임에 있는 경우만)
  3. 정책 카드 리스트
  4. 챗봇 플로팅 박스(와이어프레임에 있는 경우만)

`PolicyRecommendationCard`

- props: `{ policy, personaId, imageSrc }`
- 전체 카드 클릭: `/${personaId}/policy-detail`
- `aria-label`: `"{policy.title} 정책 상세보기"`
- 표시:
  - 정책명
  - 1~2줄 요약
  - 지원 주기/제공 형태 tag
  - 온라인 신청 가능 여부 tag
  - 우측 화살표

`PolicyFilterChip`

- 카테고리 선택 상태는 mock state로만 관리한다.
- 선택 시 실제 데이터 필터링을 하더라도 서버 호출은 없다.

## 6. 레이아웃 계획

- 목록은 1열 카드 리스트가 기본이다.
- 320px 폭에서 정책명이 긴 `국민내일배움카드제 직업훈련지원(훈련비, 훈련장려금)`도 카드 밖으로 넘치지 않게 줄바꿈한다.
- 카드 내부 긴 설명은 2~3줄 clamp 처리하되, 데이터 원문은 `title` 또는 상세 페이지에서 보존한다.
- 하단 내비게이션과 마지막 카드가 겹치지 않도록 bottom padding을 둔다.
- 플로팅 챗봇 박스가 있으면 bottom nav 위에 겹치지 않게 위치한다.

## 7. 링크 동작

- 정책 카드 클릭: `/${personaId}/policy-detail`
- 챗봇 플로팅 박스: `/${personaId}/tax-chat`
- 뒤로/상단 CTA는 와이어프레임에 있는 경우에만 구현한다.
- 외부 복지로 링크는 상세 페이지에서만 보여주고, 새 창 열기 또는 mock button으로 처리한다.
