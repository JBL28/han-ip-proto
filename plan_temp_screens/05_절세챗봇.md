# 절세챗봇 페이지 구현 계획

## 0. UI 기준

- 절세챗봇 페이지는 **메인화면을 레퍼런스**로 삼아 구현한다.
- AppShell 표준 비율, 상단 헤더, 하단 내비게이션, 카드 radius, 폰트 토큰, 색상 토큰을 유지한다.
- 채팅 UI도 별도 서비스처럼 보이지 않고 같은 앱 안의 세금 상담 화면으로 보여야 한다.
- 채팅 말풍선, 입력창, 추천 질문 카드의 높이는 모바일 터치 기준 48px 이상을 지킨다.
- 그라데이션은 금지한다. 챗봇 배경, 말풍선, 입력창은 단색 또는 기존 토큰 기반 색만 사용한다.

4. `연말정산 했는데, 종합소득세 신고 해야 하나요?`
5. `공제가 무엇인가요?`
6. `소득공제와 세액공제 차이가 뭔가요?`
7. `부업 소득 신고 해야 하나요?`
8. `주식도 세금신고 하나요?`

### 김갓생 추천 질문 3개

## 3. 데이터 원칙

- 챗봇 답변/추천 질문은 `mockdatas/index.md`의 챗봇 질문 목록과 페르소나별 흐름을 우선 사용한다.
- 실제 AI/API 호출은 구현하지 않는다.
- 입력/전송/추천 질문 클릭 같은 UI 상호작용은 mock 상태로 동작하게 한다.
- 사용자가 질문을 누르면:
  - user bubble에 질문 텍스트 추가
  - bot bubble에는 짧은 mock 응답을 표시
  - 응답 내용은 법률/금액을 새로 만들지 말고 “상세 공제 페이지에서 확인해볼게요” 수준의 안내 또는 mockdatas 문구 기반으로 제한한다.
- 파일 업로드 기능은 설계문서에 있으나 실제 파일 업로드/분석은 구현하지 않는다. 와이어프레임에 업로드 버튼이 있으면 mock UI만 표시한다.

## 4. 이미지 에셋 생성 계획

필요한 투명 배경 PNG:

- 챗봇 AI 아바타
- 사용자 기본 아바타 2종 또는 공통 실루엣
- 파일 업로드/문서 아이콘이 와이어프레임에 이미지로 필요할 경우 문서+클립 오브젝트

저장 위치:

- `src/assets/tax-chat/`

파일명:

- `ai-chat-avatar.png`
- `user-avatar-hanbeoteam.png`
- `user-avatar-kimgatsaeng.png`
- `upload-document.png`

생성 프롬프트 공통 규칙:

- 투명 배경 PNG, 텍스트 없는 작은 앱용 캐릭터/오브젝트.
- 신뢰감 있는 세금/금융 챗봇 느낌, 흰색·연파랑·네이비 중심.
- 말풍선/카드 프레임은 이미지에 넣지 않고 CSS로 구현한다.
- 이미지 안에 숫자, 한글, 로고, 실제 기관 마크를 넣지 않는다.

## 5. 컴포넌트 구조

`TaxChatPage.jsx`

- `useParams()`로 `personaId` 확인.
- `useState`로 messages, selectedSuggestion, draft input 관리.
- 데이터:
  - `chatPersonaByPersonaId`
  - `chatQuestionsByPersonaId`
  - `chatSummaryByPersonaId`
- 화면은 아래 순서로 구성한다.
  1. 상단 챗봇 제목/페르소나 badge
  2. 환급금/추천 요약 notice
  3. 메시지 리스트
  4. 자주 묻는 질문 chips/cards
  5. 하단 입력창

`ChatSummaryNotice`

- 메인 페이지 `BenefitAlertCard`보다 작은 공지 카드.
- 긴 금액 문구는 원문을 보존하고 CSS 줄바꿈으로 처리한다.

`ChatMessageBubble`

- role: `bot` | `user`
- bot 말풍선은 연파랑 단색, user 말풍선은 네이비/블루 단색 또는 outline.
- 말풍선 안 텍스트는 `ui-body-muted` 계열 또는 새 공통 typography class를 사용한다.

`ChatSuggestionChip`

- 추천 질문 전체를 button으로 만든다.
- 클릭 시 메시지 mock 추가.

`ChatInputBar`

- input placeholder는 와이어프레임 문구를 우선한다.
- 전송 버튼은 48px 터치 영역.
- 실제 서버 전송 없음.

## 6. 레이아웃 계획

- `.app-content` 하단 nav와 입력창이 겹치지 않도록 bottom padding을 충분히 둔다.
- 입력창을 sticky로 둘 경우 bottom nav 위에 위치시키고, safe area를 고려한다.
- 메시지 리스트는 세로 스크롤 흐름 안에 두며 AppShell 전체 overflow를 막지 않는다.
- 추천 질문은 1열 또는 wrap chip row로 구성하되, 320px에서 가로 스크롤이 생기지 않게 한다.
- 카드/말풍선 root에는 임의 margin을 넣지 않고 부모 gap으로 간격을 관리한다.

## 7. 링크 동작

- `AI 페르소나 변경`: `/${personaId}/persona`
- 요약 카드의 “공제 상세” 성격 링크가 있으면 `/${personaId}/deduction-detail`
- 정책 관련 질문 CTA가 있으면 `/${personaId}/policy-detail` 또는 `/${personaId}/policies`
- 실제 AI, 파일 업로드, 전화 연결, 서버 저장은 구현하지 않는다.

## 8. 검증 계획

- `npm run validate`
- 그라데이션 금지 검색:
  - `grep -R "linear-gradient\|radial-gradient\|conic-gradient" -n src dist/index.html dist/assets/*.css dist/assets/*.js || true`
- 추천 질문 클릭 시 user/bot bubble이 추가되는지 확인
- 입력 후 전송 버튼 클릭 시 mock 메시지가 추가되는지 확인
- 입력창이 하단 내비게이션과 겹치지 않는지 확인
- 와이어프레임 텍스트 누락 수동 대조
