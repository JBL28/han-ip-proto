# 알림 페이지 구현 계획

## 0. UI 기준

- 알림 페이지는 **메인화면의 AppShell 표준 비율과 리스트 카드 스타일**을 따른다.
- 알림 카드는 `AppCard` 또는 동일 radius/shadow/token 체계를 사용한다.
- 알림의 중요도는 색을 많이 늘리지 않고 단색 배지, 아이콘, 텍스트 굵기로 구분한다.
- 알림 리스트는 하단 내비게이션과 겹치지 않아야 하며 320px 폭에서 가로 스크롤이 없어야 한다.
- 그라데이션은 금지한다.

## 1. 구현 대상

- 파일: `src/features/notifications/NotificationsPage.jsx`
- 현재 placeholder를 실제 알림 화면으로 교체한다.
- 예상 컴포넌트:
  - `src/components/NotificationCards.jsx`
    - `NotificationFilterTabs`
    - `NotificationItemCard`
    - `NotificationStatusBadge`
- 와이어프레임: `wireframes/알림.png`
- 진입 경로: AppShell header의 알림 버튼 `/${personaId}/notifications`

## 2. 텍스트 준수 목록

와이어프레임의 제목, 탭, 알림 제목/본문/시간/상태 텍스트는 구현 직전 다시 수동 대조한다. 알림 문구는 `mockdatas/index.md`의 페르소나별 알림 섹션을 우선 사용한다.

### 상단/탭 후보

- `알림`
- `전체`
- `혜택 추천`
- `신청/처리 현황`

탭이 와이어프레임에 없으면 구현하지 않는다.

### 한버팀 혜택 추천 알림 2개

1. `💡 한버팀님! 부모님께 매달 이체하신 내역이 있어요. '부양가족 인적공제' 대상이 되는지 1분 만에 확인해 볼까요?`
2. `이번 달 문화누리카드 지원금 신청이 시작되었어요. 아이와 함께 쓸 수 있어요!`

### 한버팀 신청/처리 현황 2개

1. `✅ 한부모가족 아동양육비 21만 원이 계좌로 정상 입금되었습니다.`
2. `🚨 근로장려금 신청 마감이 딱 3일 남았어요! 터치 한 번으로 쉽게 신청하세요.`

### 김갓생 혜택 추천 알림 2개

1. `💡 이번 달 신용카드 지출액이 기준치를 넘었어요! 내일부터는 체크카드를 써야 연말정산 때 돈을 더 받아요.`
2. `김갓생님, 안경점에서 결제한 15만 원을 연말정산에 반영할까요? (터치하여 1분 만에 영수증 등록하기)`

### 김갓생 신청/처리 현황 2개

1. `✅ 고향사랑기부제 10만 원 납부가 완료되어, 연말정산 100% 세액공제 항목에 자동 추가되었습니다.`
2. `📝 집주인 눈치 볼 필요 없어요! 월세 현금영수증 발급 신청이 국세청에 정상 접수되었습니다.`

## 3. 데이터 원칙

- 알림 문구는 `mockdatas/index.md`의 페르소나별 알림 섹션을 그대로 사용한다.
- 알림 시간, 읽음/안읽음, 상태 라벨은 와이어프레임에 있는 경우에만 추가한다.
- 임의 알림 생성 금지.
- 실제 push, 서버 저장, 알림 읽음 영속 저장은 구현하지 않는다.
- 탭 필터/읽음 상태는 mock state로만 동작하게 한다.

### 데이터 구조 제안

```js
{
  id: 'han-parent-transfer',
  personaId: 'hanbeoteam',
  category: 'recommendation',
  text: "💡 한버팀님! 부모님께 매달 이체하신 내역이 있어요. '부양가족 인적공제' 대상이 되는지 1분 만에 확인해 볼까요?",
  status: 'unread',
  to: '/hanbeoteam/deduction-detail',
  source: 'mockdatas/index.md > 한버팀 > 알림'
}
```

## 4. 이미지 에셋 생성 계획

알림 화면은 가능하면 SVG/CSS 아이콘으로 충분하다. 와이어프레임에 큰 이미지가 필요한 경우에만 아래 PNG를 생성한다.

필요한 투명 배경 PNG 후보:

- 알림 상단 벨 일러스트
- 혜택 추천용 전구/체크 오브젝트
- 처리 완료용 체크 문서 오브젝트
- 마감 임박용 작은 캘린더/경고 오브젝트

저장 위치:

- `src/assets/notifications/`

파일명:

- `notification-bell.png`
- `benefit-tip.png`
- `status-check-document.png`
- `deadline-calendar.png`

생성 프롬프트 공통 규칙:

- 투명 배경 PNG, 텍스트 없는 단일 오브젝트.
- 이모지 문구는 텍스트 데이터에만 두고 이미지에 넣지 않는다.
- 실제 기관 로고, 숫자, 날짜 텍스트 금지.
- CSS에서 단색 배경 원/카드를 만든다.

## 5. 컴포넌트 구조

`NotificationsPage.jsx`

- `useParams()`로 `personaId` 확인.
- `useState`로 activeTab과 read state 관리.
- 페르소나별 알림 배열을 선택해 렌더링한다.
- 본문 순서:
  1. 상단 제목
  2. 탭/필터(와이어프레임에 있을 때)
  3. 알림 리스트

`NotificationFilterTabs`

- options: `전체`, `혜택 추천`, `신청/처리 현황`
- button으로 구현하고 선택 상태는 `aria-pressed` 또는 `aria-current`로 표현한다.

`NotificationItemCard`

- props: `{ notification, onClick }`
- 카드 클릭 시 해당 상세 경로로 이동하거나 읽음 상태만 바꾼다.
- `category`에 따라 좌측 작은 아이콘/배지를 다르게 표시한다.

## 6. 레이아웃 계획

- 리스트는 1열 세로 stack.
- 알림 문구가 길기 때문에 카드 내부 line-height와 padding을 충분히 둔다.
- 이모지는 텍스트 앞에 그대로 두되, 카드 아이콘과 중복되어 산만하면 CSS 아이콘은 생략한다.
- 카드 root에는 margin을 넣지 않고 부모 gap으로만 간격을 관리한다.
- 마지막 카드가 bottom nav에 가려지지 않도록 bottom padding 확보.

## 7. 링크 동작

- 공제 관련 알림: `/${personaId}/deduction-detail`
- 정책 관련 알림: `/${personaId}/policy-detail` 또는 `/${personaId}/policies`
- 마감/상태 알림: 해당 상세 페이지로 이동하거나 mock read 처리
- 실제 신청, 입금 확인, 영수증 등록은 구현하지 않는다.

## 8. 검증 계획

- `npm run validate`
- 그라데이션 금지 검색:
  - `grep -R "linear-gradient\|radial-gradient\|conic-gradient" -n src dist/index.html dist/assets/*.css dist/assets/*.js || true`
- 페르소나별 알림 4개씩 표시되는지 확인
- 탭 필터가 mock state로 동작하는지 확인
- 알림 카드 클릭 라우팅/읽음 상태 확인
- `mockdatas/index.md` 알림 문구와 대조
- 와이어프레임 텍스트 누락 수동 대조
