# 정책 상세보기 페이지 구현 계획

## 0. UI 기준

- 정책 상세보기 페이지는 **공제 상세보기와 같은 상세 페이지 패턴**을 사용한다.
- 메인화면의 AppShell, 카드 radius, 폰트 토큰, 색상 토큰, 하단 내비게이션 감각을 유지한다.
- 정책 상세는 공공서비스 정보가 많으므로 `정책명 → 요약 → 지원 정보 → 신청 정보 → 출처/문의` 순서로 읽히게 한다.
- 그라데이션은 금지한다. 배지/상태/CTA는 단색만 사용한다.

## 1. 구현 대상

## 3. 데이터 원칙

- `servNm`, `servDgst`, `jurMnofNm`, `jurOrgNm`, `sprtCycNm`, `srvPvsnNm`, `onapPsbltYn`, `rprsCtadr`, `servDtlLink` 등 XML 필드 근거를 사용한다.
- 임의 기관명, 전화번호, 링크, 신청 조건, 지원 금액을 추가하지 않는다.
- 목록 페이지에서 선택한 카드 id를 route state/query/local state로 넘기지 않는 1차 구현이면, 페르소나별 기본 정책 상세를 보여준다.
- 향후 정확한 상세 전환이 필요하면 `/:personaId/policy-detail/:policyId` route를 추가한다.

## 4. 이미지 에셋 생성 계획

필요한 투명 배경 PNG:

- 정책 상세 상단: 공공서비스 문서+체크
- 아이돌봄서비스: 아이 돌봄/보호 아이콘형 일러스트
- 통합문화이용권: 문화 티켓/카드
- 직업훈련지원: 노트북+학습 노트
- 청년월세: 집+동전

저장 위치:

- `src/assets/policy-detail/`

파일명:

- `policy-detail-document.png`
- `child-care-detail.png`
- `culture-detail.png`
- `training-detail.png`
- `rent-detail.png`

생성 프롬프트 공통 규칙:

- 투명 배경 PNG, 텍스트/숫자/기관 로고 없는 오브젝트.
- 실제 정부/복지로 로고 금지.
- 이미지 배경, 그림자, 카드 프레임은 CSS에서 처리한다.
- 흰색·연파랑·네이비 중심, 필요 시 작은 초록 체크 포인트만 사용한다.

## 5. 컴포넌트 구조

`PolicyDetailPage.jsx`

- `useParams()`로 `personaId` 확인.
- `getPersonaPolicyDetail(personaId)`로 기본 상세 데이터 선택.
- 본문 순서:
  1. 상단 정책명/요약 hero
  2. 지원 정보 field list
  3. 신청/문의 정보 field list
  4. source/link card
  5. 하단 CTA

`PolicyDetailHero`

- 정책명, 요약, 대표 이미지, 주요 tag 2~3개 표시.

`PolicyFieldList`

- `{ label, value }` 배열을 받아 표시.
- 값이 없는 필드는 렌더링하지 않는다.
- label은 와이어프레임 문구와 맞춘다.

`PolicyExternalLinkCard`

- `servDtlLink`가 있으면 `자세히 보기` CTA 표시.
- 외부 링크는 새 탭 또는 mock button 처리. 실제 신청을 완료한 것처럼 보이게 하지 않는다.

## 6. 레이아웃 계획

- 1열 상세 카드 stack.
- 긴 URL은 직접 노출하지 않고 CTA label로 처리한다.
- 긴 정책명은 줄바꿈 허용.
- 마지막 CTA는 하단 nav와 겹치지 않도록 bottom padding을 둔다.
- 공제 상세와 같은 section gap/radius/padding을 유지한다.

## 7. 링크 동작

- 뒤로가기: `/${personaId}/policies`
- `챗봇에게 물어보기`: `/${personaId}/tax-chat`
- `자세히 보기`: `servDtlLink`가 있는 경우 외부 링크로 연결하거나 프로토타입 mock button으로 표시
- 신청 완료/저장/스크랩은 실제 저장 없이 UI 상태만 처리한다.

## 8. 검증 계획

- `npm run validate`
- 그라데이션 금지 검색:
  - `grep -R "linear-gradient\|radial-gradient\|conic-gradient" -n src dist/index.html dist/assets/*.css dist/assets/*.js || true`
- XML 원문 필드 대조
- 정책명/기관/문의처/링크가 source와 일치하는지 확인
- 긴 정책명/설명이 320px에서 넘치지 않는지 확인
- 와이어프레임 텍스트 누락 수동 대조
