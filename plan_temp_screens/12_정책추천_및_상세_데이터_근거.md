# 정책/공제 데이터 근거 정리 계획

## 0. 목적

- 정책추천, 정책상세, 공제상세, 알림, 마이페이지 구현 전에 mockdatas 근거를 화면별로 연결한다.
- 임의 정책/공제/금액/일정 문구 추가를 방지한다.
- 구현자는 이 문서를 데이터 계약처럼 사용하고, UI 파일에는 source 없는 문자열을 추가하지 않는다.

## 1. 대상 소스

- `mockdatas/index.md`
  - 페르소나별 시나리오 흐름, 메인페이지 금액, 절세추천 항목, 챗봇 질문, 정책추천 후보, 마이페이지 금액, 알림 문구의 canonical source.
- `mockdatas/text/index.txt`
  - `mockdatas/index.md`를 text 변환한 사본.
- `mockdatas/text/*.txt`
  - XML/PDF 변환 텍스트.
  - 복지로 XML 필드는 정책추천/정책상세의 canonical source.
  - PDF/HWP fallback 추출은 법령 근거 보조 source.
- 원본 XML: `mockdatas/*.xml`
  - 변환 텍스트와 필드 충돌 시 원본 XML을 우선 확인한다.
- 스크립트:
  - `scripts/extract-mockdata-text.py`
  - Markdown/XML/PDF를 `mockdatas/text/`로 변환하는 로컬 스크립트.
  - 현재 환경은 `pdftotext`, `pypdf`, `pdfminer` 없이 Python stdlib fallback을 사용하므로 PDF 결과가 빈약할 수 있다.
- npm scripts:
  - `npm run dev`: Vite dev server
  - `npm run build`: Vite production build
  - `npm run preview`: Vite preview server
  - `npm run validate`: 현재 `npm run build`

## 2. 정리 방식

- 화면별로 사용할 필드와 source 파일을 표로 기록한다.
- UI에 표시하는 문구가 source와 다르면 구현 전 와이어프레임 근거를 명시한다.
- PDF/HWP fallback 추출이 비어 있는 파일은 `mockdatas/index.md`를 canonical source로 사용한다.
- 데이터 배열에는 최소한 `title`, `description`, `source`를 둔다.
- 금액/혜택/정책/공제/일정은 source 없이 추가하지 않는다.

## 6. 이미지 에셋 데이터 연결 원칙

- 이미지 파일명은 데이터 id와 느슨하게 맞춘다.
  - `child-care-policy.png` → `아이돌봄서비스`
  - `culture-card.png` → `통합문화이용권`
  - `job-training.png` → `국민내일배움카드제 직업훈련지원`
  - `youth-rent.png` → `청년월세 지원사업`
- 이미지에는 source 문구를 새겨 넣지 않는다.
- 텍스트/숫자/기관 로고가 들어간 이미지는 재생성한다.
- 이미지가 없어도 정책/공제 데이터 자체를 대체 문구로 바꾸지 않는다.

## 7. 구현 전/후 검증 방식

구현 전:

- `mockdatas/text/manifest.md`에서 source 파일 존재 여부 확인.
- `scripts/extract-mockdata-text.py`가 필요한 경우 실행해 text 산출물을 갱신한다.
- `package.json` script 확인:
  - 검증은 `npm run validate`가 기준이다.

구현 후:

- 데이터 배열의 모든 `title`, `amount`, `description`, `schedule`, `policy`, `deduction`, `notification`에 `source` 필드가 있는지 확인한다.
- source 없는 mock 문자열은 추가하지 않는다.
- `npm run validate`
- 그라데이션 금지 검색:
  - `grep -R "linear-gradient\|radial-gradient\|conic-gradient" -n src dist/index.html dist/assets/*.css dist/assets/*.js || true`
- 와이어프레임 텍스트 누락/변형 수동 대조.
