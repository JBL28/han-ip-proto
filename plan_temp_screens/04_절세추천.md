# 절세추천 페이지 구현 계획

## 0. UI 기준

- 절세추천 페이지는 **메인화면을 레퍼런스**로 삼아 구현한다.
- 메인화면에서 이미 정리한 AppShell 비율, 카드 radius, 카드 간격, 폰트 토큰, 색상 토큰, 버튼 비율, 하단 내비게이션 감각을 유지한다.
- 새 UI를 독립적으로 예쁘게 만드는 것이 아니라, 사용자가 같은 앱 안의 다음 화면으로 인식할 수 있도록 통일감을 우선한다.
- 큰 카드/배너는 가능하면 기존 `AppCard`, `ui-*` typography class, `src/components/` 컴포넌트 구조를 재사용한다.
- 개별 요소마다 임의 `font-size`, `font-weight`, `color`, margin을 새로 찍지 않는다. 필요하면 기존 토큰/컴포넌트 규칙을 먼저 따른다.
- 그라데이션은 금지한다. 배경은 단색 또는 기존 토큰 기반 색만 사용한다.

## 1. 구현 대상

- 파일: `src/features/taxSaving/TaxSavingPage.jsx`
- 현재 placeholder를 실제 절세추천 화면으로 교체한다.
- 예상 컴포넌트:
  - `src/components/TaxSavingCards.jsx`
    - `TaxSavingBenefitCard`
    - `TaxSavingSupportBanner`
  - 필요 시 페이지 전용 데이터는 `src/data/personaMockData.js`에 추가한다.
- 와이어프레임: `wireframes/절세 추천.png`
- AppShell은 `AppRoutes.jsx`에서 이미 감싸고 있으므로 페이지 파일은 본문 UI만 담당한다.

## 3. 데이터 원칙

- 화면 텍스트는 사용자가 제공한 레퍼런스 이미지의 문구를 하나하나 준수한다.
- 단, 프로젝트 지침상 금액/혜택/정책 문구는 `mockdatas/`와 충돌하지 않아야 한다.
- 구현 전 `mockdatas/index.md`와 대조해 근거가 있는 항목은 source 주석 또는 데이터 필드로 연결한다.
- JSX 내부에 임의로 텍스트를 흩뿌리지 말고, 페이지 상단의 데이터 배열 또는 `personaMockData.js` 구조로 모은다.
- 임의 금액, 임의 정책/공제명, 임의 설명은 추가하지 않는다.

### mockdatas 연결 방식

- 와이어프레임 6개 카드는 발표용 고정 시안 문구이므로, 화면 카드 텍스트는 위 문구를 우선한다.

  - `따로 사는 부모님 인적공제` → `한버팀_3번_소득세법_50`, `한버팀_3번_소득세법_53`

- 그라데이션 느낌의 배경판이나 빛 번짐을 넣지 않는다.

## 5. 컴포넌트 구조

`TaxSavingPage.jsx`

- `useParams()`로 `personaId` 확인.
- `taxSavingCards` 배열 정의 또는 `getPersonaTaxSavingData(personaId)` 사용.
- 본문 순서:
  1. 상단 제목 영역
  2. 2열 추천 카드 그리드
  3. 하단 지원금 배너

`TaxSavingBenefitCard`

- props: `{ title, amount, tag, imageSrc, to, source }`
- root는 `Link` 또는 `article + Link`로 구성한다.
- 카드 전체 터치 영역은 48px 이상이며, `aria-label`은 `"{title} {amount} {tag} 상세보기"`로 둔다.
- 카드 내부: 이미지, 제목, 금액, 태그, 우측 화살표.

`TaxSavingSupportBanner`

- props: `{ title, lines, ctaLabel, imageSrc, to }`
- 좌측 텍스트/버튼, 우측 투명 배경 이미지.
- 배너 배경은 단색 연파랑 토큰 계열만 사용.

## 6. 레이아웃 계획

- AppShell 비율은 기존 `app-shell--standard`를 그대로 사용한다.
- `.app-content`가 하단 내비게이션 위에서 스크롤 가능해야 한다.
- 페이지 root는 `main2-page`와 유사한 세로 stack/gap 리듬을 유지한다.
- 카드 그리드는 2열이며, 320px 폭에서도 카드 내부 텍스트가 줄바꿈되어 가로 스크롤이 없어야 한다.
- 카드 root에는 margin을 넣지 않고, grid gap만 사용한다.
- 이미지 박스는 카드 상단 또는 좌상단에 고정하고, 이미지 자체는 `alt="" aria-hidden="true"`로 둔다.
- 하단 배너는 1개 CTA만 둔다.

## 7. 스타일 규칙

- 메인화면에서 쓰는 카드/텍스트/버튼 규칙과 통일한다.
- 텍스트는 기존 `ui-*` typography class 우선 사용:
  - 제목: `ui-section-title` 또는 새 공통 class 추가
  - 본문/설명: `ui-body-muted`
  - 금액: `ui-amount`, `ui-amount-unit`
  - CTA: `ui-action`
- 새 텍스트 계층이 정말 필요하면 `global.css`에 token/class를 먼저 추가하고, 요소 selector에 직접 폰트 값을 쓰지 않는다.
- 색상은 기존 `--color-text-*`, `--accent`, `--surface`, `--line` 계열 토큰을 우선 사용한다.
- 카드 간 외부 여백은 부모 grid/flex gap에서만 관리한다.
- 그라데이션 금지.

## 8. 링크 동작

- 추천 카드 클릭: `/${personaId}/deduction-detail`
- 하단 `내 맞춤 혜택 찾기`: `/${personaId}/policies`
- `personaId`가 없을 때는 `getRememberedPersonaId()` fallback을 사용한다.
- 실제 신청/계산 로직은 만들지 않는다.
