# 와이어프레임 공통 구현 원칙

## 0. UI 기준

- 모든 화면은 **메인화면을 레퍼런스**로 삼아 구현한다.
- AppShell 비율, 상단 헤더, 하단 내비게이션, 카드 radius, 카드 간격, 폰트 토큰, 색상 토큰, 버튼 비율은 메인화면과 통일한다.
- 독립적으로 새 디자인을 만들지 않고, 같은 앱 안의 화면 전환으로 느껴지도록 일관성을 유지한다.
- 재사용 UI 컴포넌트는 `src/components/`에 만든다.
- feature 폴더에는 화면 page와 해당 feature 전용 데이터/상태 로직만 둔다.
- 그라데이션은 금지한다. 단색 배경과 기존 토큰만 사용한다.

## 1. 텍스트 준수 원칙

- 와이어프레임의 텍스트는 구현 전 화면별 계획 파일에 체크리스트로 옮긴 뒤 한 줄씩 대조한다.
- 사용자가 제공한 와이어프레임 문구를 임의로 바꾸거나 요약하지 않는다.
- mockdatas에 근거가 필요한 금액/혜택/정책/공제/일정 데이터는 `mockdatas/`, `mockdatas/text/`와 대조한다.
- mockdatas에 없는 데이터는 와이어프레임 표시 문구로만 취급하고, 구현 파일에 source 주석을 남긴다.

## 2. 컴포넌트/스타일 원칙

- 큰 카드/배너는 기존 `AppCard` 사용을 우선한다.
- 버튼은 기존 `SignupActionBar`, `EqualButtonGroup`, `main2-primary-cta`, `ui-action` 패턴을 우선 재사용한다.
- 텍스트는 `ui-*` typography class를 우선 사용한다.
- 요소별로 `font-size`, `font-weight`, `color`를 새로 흩뿌리지 않는다.
- 카드 바깥 여백은 부모 `gap`에서만 관리한다. 카드 root에 임의 margin을 넣지 않는다.
- 이미지가 필요한 화면은 투명 배경 PNG를 생성해 `src/assets/<feature>/`에 저장한다.

## 3. 검증 공통

- `npm run validate`
- 그라데이션 금지 검색: `grep -R "linear-gradient\|radial-gradient\|conic-gradient" -n src dist/index.html dist/assets/*.css dist/assets/*.js || true`
- 와이어프레임 텍스트 누락/변형 수동 대조
- 메인/회원가입/AppShell 비율 회귀 확인
